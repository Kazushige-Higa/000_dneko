コラムAPI 新規作成 設定手順

microCMSに column APIを新規作成し、ブログと同じ構成（カテゴリタブ付き）でコラム記事を管理できるようにします。

完成イメージ（トップページ・一覧ページ共通）：

すべて ビジネス デザイン マーケティング

STEP 1：column APIを新規作成

1 microCMS管理画面 → 左上「＋」ボタン（API追加）→「自分で決める」
項目設定値
API名コラム
エンドポイントcolumn
APIの型リスト形式
重要：エンドポイントは必ず column にしてください。PHPコード側でこのエンドポイント名を使用しています。

STEP 2：APIスキーマを設定

2 作成した「コラム」API → 右上「API設定」→「APIスキーマ」タブで以下のフィールドを追加

ブログAPIと同じフィールド構成にします：

フィールドID表示名種類必須備考
titleタイトルテキストフィールドはい記事のタイトル
content本文リッチエディタはい記事の本文
thumbnailサムネイル画像いいえ一覧表示用の画像
description概要テキストエリアいいえ一覧ページの説明文
categoryカテゴリセレクトフィールドいいえタブ分け用（下記参照）
3 category セレクトフィールドの選択肢を追加（例）：
値（value）表示ラベル
ビジネスビジネス
デザインデザイン
マーケティングマーケティング
ライフスタイルライフスタイル
ポイント：選択肢は後からいつでも追加・削除できます。カテゴリ名がそのままタブのラベルとして表示されます。カテゴリ未設定の記事は「その他」タブに自動分類されます。

STEP 3：オプションフィールド（ブログと同じカスタムフィールドを追加する場合）

ブログと同様に、以下のカスタムフィールドも追加できます（任意）：

フィールドID表示名種類用途
faqよくあるご質問繰り返しFAQ アコーディオン
ctaCTAボックス繰り返し商品ランキング・アフィリエイト
speech吹き出し繰り返しリード文の吹き出し
補足：これらのカスタムフィールドの詳細な設定手順は、それぞれの設定手順書を参照してください。フィールドIDとサブフィールドの構成がブログAPIと同じであれば、entry.php側の表示コードがそのまま使えます。

STEP 4：APIスキーマ完成イメージ

column API（新規作成）
├─ title（テキストフィールド）← 記事タイトル
├─ content（リッチエディタ）← 本文
├─ thumbnail（画像）← サムネイル
├─ description（テキストエリア）← 概要
├─ category（セレクトフィールド）← カテゴリタブ用
│    選択肢: ビジネス / デザイン / マーケティング / ライフスタイル ...
│
├─ faq（繰り返し）← よくあるご質問 ※任意
├─ cta（繰り返し）← CTAボックス ※任意
└─ speech（繰り返し）← 吹き出し ※任意
